## 1. プロジェクト概要

**Abro** は、留学・ワーホリを計画・実行・記録するための AI 駆動プラットフォーム。
旅行AIプランナーの「Mindtrip」を留学・ワーホリ文脈に翻訳し、SNS・コミュニティ・AI自動ブックマーク機能を統合したサービス。

### ミッション
留学・ワーホリの「準備」「出会い」「実行」をすべて AI で動詞化する。

### ターゲット
- 留学・ワーホリ希望者(検討層・準備層・渡航中層・帰国層の4フェーズ)
- 留学エージェント
- 留学生の親
- 現地事業者(語学学校、レストラン、ツアー会社など)

---

## 2. 4つの柱

すべての機能は以下4本柱のどれかに属する。実装時は「これはどの柱?」を常に意識すること。

| 柱 | 役割 | 主機能 |
|---|---|---|
| **Plan** | AIで留学プランを作る | AIチャット、費用シミュレーター、エージェント共同編集 |
| **Prepare** | 出発までの情報・タスクを集約 | AI自動分類ブックマーク、タスク管理、親アカウント |
| **Connect** | 人とつながる | SNS、マッチング、コミュニティ、掲示板、エージェントルーム |
| **Act** | 予約・契約・現地生活を実行 | 予約フロー、家計簿、緊急サポート、クリエイター収益 |

---

## 3. ユーザーフェーズ(時間軸の4層)

ユーザーは時間経過とともに以下のフェーズを移動する。
**重要:データモデルとUIは「現在のフェーズ」を常に意識する。**

| Lv | フェーズ | 状態 | 主な行動 |
|---|---|---|---|
| 1 | 検討層 | 留学を考え始めた | 情報収集、先輩への質問 |
| 2 | 準備層 | エージェント面談中・申込済 | プラン作成、書類準備 |
| 3 | 渡航中層 | 現地で生活中 | コミュニティ参加、生活管理 |
| 4 | 帰国層 | 帰国後 | クリエイター化、就活 |

各ユーザーは `users.phase` カラムでフェーズを管理。UIは現在のフェーズに応じて出し分ける。

## 4. 技術スタック

### フロントエンド
- **React Native (Expo)** — クロスプラットフォーム(iOS/Android)
- **TypeScript** — 型安全性
- **Expo Router** — ファイルベースルーティング
- **NativeWind (Tailwind CSS for RN)** — スタイリング
- **Zustand** — グローバル状態管理
- **React Query (@tanstack/react-query)** — サーバー状態管理
- **react-native-reanimated** — アニメーション
- **expo-image** — 画像最適化

### バックエンド
- **Supabase** — Postgres + Auth + Storage + Edge Functions + Realtime
- **Supabase Auth** — メール/Google/Apple/LINE ログイン
- **Supabase Realtime** — チャット、通知、リアルタイム更新
- **Supabase Storage** — 画像・PDF・動画
- **Supabase Edge Functions (Deno)** — AI 呼び出し、外部 API 連携
- **pgvector** — ブックマークのベクトル検索(AI 自動分類)

### AI
- **Anthropic Claude API** — チャット、自動分類、プラン生成
- **OpenAI Embeddings (text-embedding-3-small)** — ベクトル化
- 将来:画像認識用に Claude のマルチモーダル、または GPT-4 Vision

### 外部 API
- **Skyscanner / Amadeus** — 航空券
- **Google Places API** — 地図・スポット情報
- **OpenAI / Anthropic** — AI 機能
- **LINE Messaging API** — LINE 共有・通知
- **Stripe** — 決済(Premium プラン、エージェント課金)
- **Wise API** — 送金・家計簿(将来)
- **GetYourGuide / Viator** — 現地体験(アフィリエイト)

### インフラ
- **Vercel** — Edge Functions のホスティング(必要なら)
- **EAS Build (Expo)** — ネイティブビルド
- **EAS Submit** — App Store / Google Play 提出
- **Sentry** — エラー監視
- **PostHog** — 行動分析

## 7. 開発の原則

### コード品質
1. **TypeScript strict mode**:`any` 禁止、型は明示
2. **コンポーネントは小さく**:1ファイル200行以内を目安
3. **ロジックとUIを分離**:`hooks/` にロジック、`components/` にUI
4. **エラーハンドリング必須**:try/catch + ユーザーへのフィードバック
5. **アクセシビリティ**:`accessibilityLabel` を全インタラクティブ要素に

### Supabase
1. **RLS (Row Level Security) 必須**:全テーブルで有効化
2. **マイグレーションファイル**:すべての DB 変更は `supabase/migrations/` に
3. **Edge Functions**:外部 API キーはサーバー側で隠蔽
4. **インデックス**:検索カラムには必ず

### AI
1. **プロンプトはテンプレ化**:`lib/ai/prompts.ts` に集約
2. **ストリーミング対応**:長文応答は Stream で
3. **エラー時のフォールバック**:AI が落ちてもアプリが死なない
4. **トークン管理**:長期会話は要約してコスト削減

### UI/UX 哲学(Mindtrip から学んだもの)
1. **トップは超ミニマル**:挨拶 + アクションチップ + チャット入力欄のみ
2. **常時固定パネルは作らない**:すべて文脈起動型
3. **チャットがホーム**:全機能はチャット経由でも到達可能
4. **5タブ構造**:Chats / Plan / Explore / Saved / Me

## 8. 国際化(i18n)

- 初期は **日本語** のみ
- Phase 2 で **英語** 対応
- すべての文字列は `lib/i18n/` 経由(直接ハードコードしない)

## 9. セキュリティ・プライバシー

- 親アカウント連携は本人の明示的同意が必須
- 未成年ユーザーは特別扱い(別フロー)
- ビザ・パスポート情報は暗号化保存
- 位置情報は精密度を選択可能(街単位 / 区単位 / 正確)

## 11. 重要な制約・哲学

- **エージェントとユーザーは対等な共同編集者**(エージェントが上ではない)
- **AI はガードレール付き**(自前データを優先、汎用知識はフォールバック)
- **データは長期蓄積を前提**(留学計画は1年以上)
- **親の存在を常に意識**(未成年・若年層の親は最大の意思決定者)
